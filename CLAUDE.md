# 配送記録アプリ 『Truck Tracker』

## 概要
- 自社の配送部門の配達記録を管理するwebアプリ
- 各ドライバーがスマホから「出発」や「配達先に到着」などのアクションの前に打刻
- PC/タブレット/スマホ レスポンシブデザイン
- 登録データは最終的に販売管理のデータと結合して、利益の測定に使用する
- リアルタイムで車両がどこに配達中か把握（PC想定）
- 日付範囲指定してCSVダウンロード（PC想定）
- 日報データ編集（PC想定）


## 環境
- Supabase
- netlify
- html/css/js
- Bootstrap

## 構成
- index.html	:日報記録用（スマホ想定）
- dashboard.html:管理者用ダッシュボード（PC・タブレット想定）
	- 日報メンテナンス
	- CSVダウンロード
	- マスタ管理

## 想定マスタ件数
- 配送コース：100件 （courses）  
	- 配達先/曜日/支店
- 配達先：100件（destinations）
- 車輌：100台（trucks）
- 支店：10件 （branchs）

## その他
- 配達は基本的にはコースの順番通りだが、順番が変わる場合やスキップする可能性がある
- ブラウザをいったん終了→再度表示した場合は、車輌のステータスに応じた表示を行う
	- 例えば、「出発」クリック後にブラウザ終了→再度表示したら、配達先は変更不可でボタンは「到着」になっている
- 出庫時や、荷下ろし後の「次の配達先」は、配送コースで設定されている配達先からプルダウン等で１つ選択する
- 自社出庫時のoddは前回帰社時のoddを使用する
- 出発してから到着するまで1時間以上あくこともあるので、localStorage等の利用必須
- PCからもリアルタイムにモニターするので、出発や到着等、アクションごとに状態をDBに更新する
